02 · AI in Production

Features real users touch, covered by evals

Five AI features shipped into the platform for end users, plus the eval pipeline that measures them. Every AI output is reviewable - accept, edit, dismiss.

5 AI features in the onboarding platform RAG · evals · prompt design

Ask FDE (RAG-powered Q&A over specs and meetings, with source citations), meeting-to-account matching, spec generation, Jira ticket creation, and weekly update drafting. The weekly-update feature I built and shipped solo, end-to-end - spec, prompt design, evals, UI, and deployed code. An LLM-as-Judge pipeline surfaced that the RAG assistant's grounding averaged 46.7%, driven by open-ended summary prompts scoring 0–40% versus 95%+ on specific questions; I prioritized the citation prompt fix.

Collective impact: 5–7 hrs/week returned per onboarding analyst · ~4 hrs saved per new distributor on spec + ticket generation
LLM-as-Judge eval pipeline evals · hallucination detection

Designed an eval pipeline across all 5 AI features that turned grounding into a measurable, monitorable metric - catching where a RAG assistant looked fine in a demo but failed on open-ended prompts. This is the AI PM discipline: not shipping AI, but knowing when it's wrong.
